## Story

**As a** CLI user processing scanned PDFs,
**I want** to control OCR behavior through CLI flags and pipeline configuration,
**so that** I can customize language, resolution, and enable/disable OCR as needed.

## Scope

**IN:**
- PipelineConfig extension com bloco `ocr`
- CLI flags: `--no-ocr`, `--ocr-language`, `--ocr-dpi`
- Progress reporting por página durante OCR
- Propagação de config OCR pelo pipeline

**OUT:**
- Core OCR engine — Story 1.7
- Quality scoring — Story 1.9
- Cloud OCR provider config — future story

## Complexity
**Story Points:** 3 (S) — config propagation e CLI flags, sem lógica complexa

## Acceptance Criteria

1. [x] `PipelineConfig` extendido com bloco `ocr` (enabled, language, dpi, concurrency)
2. [x] CLI flag `--no-ocr` desabilita fallback OCR (mantém comportamento atual: erro em PDF escaneado)
3. [x] CLI flag `--ocr-language <lang>` configura idioma Tesseract (default: 'eng')
4. [x] CLI flag `--ocr-dpi <number>` configura resolução de rasterização (default: 300)
5. [x] Progress callback reporta progresso por página durante OCR ("OCR page 3/15...")
6. [x] Preset configs propagam OCR settings corretamente
7. [x] Testes unitários para CLI parsing dos novos flags
8. [x] Testes para PipelineConfig com OCR options
9. [x] `npm run build`, `npm test`, `npm run lint`, `npm run typecheck` passam sem erros
