## Story

**As a** developer building the EXTRACTA pipeline,
**I want** a structure builder that reconstructs the document's semantic hierarchy from clean HTML,
**so that** the output preserves chapters, sections, and subsections with proper Markdown heading levels and rich metadata.

## Acceptance Criteria

1. [x] Detecta headings por: tags HTML (h1-h6), numeração (1., 1.1, I., a.), caixa alta, padrões linguísticos
2. [x] Reconstrói hierarquia como árvore: Document → Chapter → Section → Subsection → Content
3. [x] Gera Markdown estruturado com heading levels corretos (`#`, `##`, `###`)
4. [x] Preserva: listas, tabelas, code blocks, blockquotes
5. [x] Gera metadados por seção: `{ chapter, section, level, word_count, charCount }`
6. [x] Extrai metadados do documento: `{ title, author, language }` (best-effort)
7. [x] Language detection automática (PT-BR, EN, ES mínimo)
8. [x] Testes com pelo menos 5 documentos com estruturas hierárquicas diferentes

## Tasks / Subtasks

- [x] **Task 1: Heading detector** (AC: 1)
  - [x] Criar `src/pipeline/heading-detector.ts`
  - [x] Detectar headings por tags HTML: `<h1>` a `<h6>` com mapeamento direto de nível
  - [x] Detectar headings por numeração: `1.`, `1.1`, `1.1.1`, `I.`, `II.`, `a)`, `A.`
  - [x] Detectar headings por caixa alta: linhas curtas (< 100 chars) em ALL CAPS
  - [x] Detectar headings por padrões linguísticos: "Chapter", "Capítulo", "Section", "Seção", "Part", "Parte"
  - [x] Scoring system para confidence de cada detecção
  - [x] Resolver conflitos quando múltiplos sinais indicam níveis diferentes

- [x] **Task 2: Hierarchy builder** (AC: 2)
  - [x] Criar `src/pipeline/structure-builder.ts` implementando `IPipelineStage<string, StructuredDocument>`
  - [x] Construir árvore hierárquica: Document (root) → Chapter (level 1) → Section (level 2) → Subsection (level 3+)
  - [x] Atribuir `id` único por seção: formato `ch{N}-s{N}-ss{N}` (e.g., "ch1-s2-ss1")
  - [x] Graceful degradation: documentos sem hierarquia detectada viram single-section
  - [x] Handle deeply nested structures (> 6 levels) colapsando para 6 max
  - [x] Conteúdo entre headings é atribuído à seção do heading anterior

- [x] **Task 3: Markdown generator** (AC: 3, 4)
  - [x] Converter HTML limpo para Markdown por seção
  - [x] Heading levels: `#` (chapter), `##` (section), `###` (subsection), etc.
  - [x] Preservar listas (`<ul>`, `<ol>` → `- `, `1. `)
  - [x] Preservar tabelas (`<table>` → Markdown table syntax)
  - [x] Preservar code blocks (`<pre><code>` → ``` fenced blocks)
  - [x] Preservar blockquotes (`<blockquote>` → `> `)
  - [x] Preservar emphasis (`<em>` → `*`, `<strong>` → `**`)
  - [x] Preservar links (`<a href>` → `[text](url)`)

- [x] **Task 4: Metadata extractor** (AC: 5, 6)
  - [x] Gerar `SectionMetadata` para cada seção: chapter, section, wordCount, charCount
  - [x] Extrair document-level metadata (best-effort):
    - `title`: primeiro heading do documento ou metadata do extractor
    - `author`: buscar padrões "by Author", "Author:" no início do documento
    - `language`: via language detection (Task 5)
  - [x] Calcular `QualityIndicators` iniciais: `structurePreserved`, `metadataCompleteness`

- [x] **Task 5: Language detection** (AC: 7)
  - [x] Implementar language detection leve (sem dependência pesada)
  - [x] Suporte mínimo: PT-BR, EN, ES
  - [x] Estratégia: análise de stopwords frequency (top 50 stopwords por idioma)
  - [x] Retornar ISO 639-1 code (e.g., "pt", "en", "es")
  - [x] Fallback: "unknown" quando confidence < threshold

- [x] **Task 6: Pipeline integration** (AC: 2, 3)
  - [x] Integrar Structure Builder no `PipelineRunner`
  - [x] Sequência atualizada: Extract → Normalize → Clean → Structure
  - [x] Input: CleanHTML (string), Output: `StructuredDocument`
  - [x] Substituir stub do pipeline por implementação real

## Dev Notes

### Architecture References

Todas as interfaces seguem o documento `docs/architecture/pipeline-architecture.md`.

**Key Interfaces:**

```typescript
// src/core/document.ts (definida em Story 1.1)
interface StructuredDocument {
  metadata: DocumentMetadata;
  sections: Section[];
  qualityIndicators: QualityIndicators;
}

interface Section {
  id: string;              // e.g., "ch1-s2-ss1"
  level: number;           // 1 = chapter, 2 = section, 3 = subsection
  title: string;
  content: string;         // Markdown content
  children: Section[];
  meta: SectionMetadata;
}

interface SectionMetadata {
  chapter?: string;
  section?: string;
  wordCount: number;
  charCount: number;
}

interface QualityIndicators {
  structurePreserved: number;  // 0-100
  noiseRemoved: number;        // 0-100
  metadataCompleteness: number; // 0-100
}
```

**Pipeline Stage:**

```typescript
// Structure Builder implementa:
IPipelineStage<string, StructuredDocument>
// Input: CleanHTML (string), Output: StructuredDocument
```

**Data Flow:**
```
CleanHTML → [Structure Builder] → StructuredDocument (tree + metadata + Markdown)
```

**Depends on:** Story 1.1 (interfaces, document types), Story 1.2 (extractors), Story 1.3 (normalizer/cleaner produzem CleanHTML)
