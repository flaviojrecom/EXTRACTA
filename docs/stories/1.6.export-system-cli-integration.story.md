## Story

**As a** developer building the EXTRACTA pipeline,
**I want** a complete export system with presets, quality scoring, cache, and full CLI integration,
**so that** the pipeline is fully functional end-to-end, producing AI-ready output files with quality metrics and a polished CLI experience.

## Acceptance Criteria

1. [ ] **Exporters:** Markdown (.md), JSON estruturado (.json), JSONL para fine-tuning (.jsonl), TXT limpo (.txt)
2. [ ] **Presets:** `knowledge-base` (hierarquia completa, sem chunking), `rag` (chunking otimizado, metadados completos), `fine-tuning` (JSONL com pares estruturados)
3. [ ] **Quality Score (0-100):** Calcula score baseado em: % estrutura preservada, consistência de chunks, completude de metadados
4. [ ] **CLI completa:**
   - `extracta process <file> --preset rag --output ./out`
   - `extracta process <file> --format md,json --chunk-size 750`
   - `extracta process <dir> --preset rag` (batch mode — processa todos os arquivos suportados)
   - `--no-cache` flag para forçar reprocessamento
   - `--verbose` para logging detalhado
   - `--dry-run` para análise sem exportação
5. [ ] **Cache:** StandardHTML intermediário cached por SHA-256 hash do arquivo (filesystem)
6. [ ] **Progress output:** Mostra etapa atual e % de progresso no terminal
7. [ ] Pipeline completo end-to-end funciona: file → analyze → extract → normalize → clean → structure → chunk → export
8. [ ] Testes E2E com pelo menos 3 documentos reais (PDF, EPUB, TXT)
9. [ ] README atualizado com exemplos de uso e documentação completa da CLI

## Tasks / Subtasks

- [ ] **Task 1: Implementar exporters** (AC: 1)

  - [ ] Criar `src/exporters/txt.ts` implementando `IExporter` — gera texto limpo sem formatação
  - [ ] Todos os exporters escrevem arquivos no `outputDir` especificado
  - [ ] Naming convention: `{original_name}.{format}` (e.g., `document.md`, `document.json`)

- [ ] **Task 2: Preset system** (AC: 2)
  - [ ] Implementar lógica de presets no pipeline runner
  - [ ] `knowledge-base`: cleaning level = light, sem chunking, export = md, incluir hierarquia completa
  - [ ] `rag`: cleaning level = standard, chunking com overlap semântico, export = json, metadados completos por chunk
  - [ ] `fine-tuning`: cleaning level = aggressive, chunking por seção, export = jsonl, pares estruturados
  - [ ] Preset configura automaticamente: cleaningLevel, chunkOptions, exportFormats, includeMetadata
  - [ ] Usuário pode override qualquer opção do preset via CLI flags

- [ ] **Task 3: Quality scorer** (AC: 3)
  - [ ] Criar `src/quality/scorer.ts`
  - [ ] Calcular score 0-100 baseado em 3 dimensões:
    - **Estrutura preservada (40%):** % de headings detectados, profundidade da hierarquia
    - **Consistência de chunks (30%):** % de chunks dentro do target range, variância de tamanho
    - **Completude de metadados (30%):** % de campos preenchidos (title, author, language, chapter, section)
  - [ ] Retornar score geral + breakdown por dimensão
  - [ ] Incluir score no output de cada exportação (`ExportResult.qualityScore`)

- [ ] **Task 4: Cache system** (AC: 5)
  - [ ] Criar `src/cache/file-cache.ts`
  - [ ] Cache key: SHA-256 hash do conteúdo do arquivo input
  - [ ] Cache location: `.extracta-cache/` no diretório de output ou temp dir
  - [ ] Estrutura: `.extracta-cache/{hash}/standard.html`, `metadata.json`, `analysis.json`
  - [ ] `get(hash: string): CachedResult | null` — retorna cache hit ou null
  - [ ] `set(hash: string, data: CachedResult): void` — salva no cache
  - [ ] `--no-cache` flag bypassa o cache completamente
  - [ ] Cache checkpoint: após extraction (StandardHTML é o item mais caro de reprocessar)

## Dev Notes

### Architecture References

Todas as interfaces seguem o documento `docs/architecture/pipeline-architecture.md`.

**Key Interfaces:**

```typescript
// src/exporters/types.ts (definida em Story 1.1)
interface IExporter {
  format: ExportFormat;
  export(doc: StructuredDocument, chunks: Chunk[], options: ExportOptions): Promise<ExportResult>;
}

type ExportFormat = 'md' | 'json' | 'jsonl' | 'txt';

interface ExportOptions {
  outputDir: string;
  preset: Preset;
  includeMetadata: boolean;
}

type Preset = 'knowledge-base' | 'rag' | 'fine-tuning';

interface ExportResult {
  files: string[];         // Paths of created files
  qualityScore: number;    // 0-100
  stats: ExportStats;
}

interface ExportStats {
  totalChunks: number;
  totalTokens: number;
  avgChunkSize: number;
  processingTimeMs: number;
}
```

**Pipeline Config:**

```typescript
interface PipelineConfig {
  cleaningLevel: 'light' | 'standard' | 'aggressive';
  chunkOptions: ChunkOptions;
  exportFormats: ExportFormat[];
  preset: Preset;
  cache: boolean;
  verbose: boolean;
}
```

**Cache Strategy:**

```
Cache key: SHA-256 hash of file content
Cache location: .extracta-cache/
Cache structure:
  .extracta-cache/{hash}/
    standard.html     # Cached StandardHTML
    metadata.json     # Cached document metadata
    analysis.json     # Cached file analysis
Invalidation: --no-cache flag forces full reprocessing
```

**Data Flow per Preset:**

```
knowledge-base: File → Extract → Normalize → Clean(light) → Structure → Export(md)
rag:            File → Extract → Normalize → Clean(standard) → Structure → Chunk → Export(json)
fine-tuning:    File → Extract → Normalize → Clean(aggressive) → Structure → Chunk → Export(jsonl)
```

**Depends on:** Story 1.1 (interfaces, CLI skeleton), Story 1.2 (extractors), Story 1.3 (normalizer/cleaner), Story 1.4 (structure builder), Story 1.5 (chunker)
