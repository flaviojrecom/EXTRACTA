## Story

**As a** developer building the EXTRACTA pipeline,
**I want** normalization and semantic cleaning stages that remove noise and standardize text,
**so that** downstream stages (structure builder, chunker) receive clean, consistent HTML without artifacts from the source format.

## Acceptance Criteria

1. [x] **Normalizer:** Padronização de encoding (UTF-8), remoção de quebras artificiais de linha, correção de hifenização, normalização Unicode, remoção de múltiplos espaços
2. [x] **Cleaner:** Remove headers/footers repetidos, numeração de página, ISBN, créditos legais
3. [x] Detecção de padrões repetitivos por frequência de ocorrência e posição no documento
4. [x] Configuração de agressividade: `light` (mínima), `standard` (recomendada), `aggressive` (máxima)
5. [x] Cada remoção é loggada para auditoria (o que foi removido e por quê)
6. [x] Testes com documentos reais que contêm ruído típico
7. [x] Pipeline integrado: Extract → Normalize → Clean funciona end-to-end

## Tasks / Subtasks

- [x] **Task 1: Implementar Normalizer stage** (AC: 1)
  - [x] Criar `src/pipeline/normalizer.ts` implementando `IPipelineStage<string, string>` (StandardHTML → NormalizedHTML)
  - [x] Encoding normalization: garantir UTF-8, remover BOM, substituir caracteres inválidos
  - [x] Whitespace cleanup: colapsar múltiplos espaços, normalizar tabs, remover trailing spaces
  - [x] Correção de hifenização: detectar palavras quebradas por hífem no fim de linha e reconectar
  - [x] Unicode normalization: NFC form, substituir variantes Unicode por equivalentes padrão (e.g., smart quotes → straight quotes)
  - [x] Remoção de quebras artificiais de linha (mid-sentence line breaks típicos de PDF)
  - [x] Preservar quebras intencionais (parágrafos, listas, code blocks)

- [x] **Task 2: Implementar Cleaner stage** (AC: 2, 3, 4)
  - [x] Criar `src/pipeline/cleaner.ts` implementando `IPipelineStage<string, string>` (NormalizedHTML → CleanHTML)
  - [x] Detecção de headers/footers repetidos: analisar texto em posições top/bottom de páginas, identificar padrões que se repetem em > 50% das páginas
  - [x] Remoção de numeração de página: detectar padrões numéricos isolados (e.g., "42", "- 42 -", "Page 42")
  - [x] Remoção de ISBN: regex para ISBN-10 e ISBN-13
  - [x] Remoção de créditos legais: detectar blocos "Copyright", "All rights reserved", etc.
  - [x] Implementar 3 níveis de agressividade:
    - `light`: apenas numeração de página e whitespace excessivo
    - `standard`: + headers/footers repetidos, ISBN, créditos legais

  - [x] Retornar lista de padrões detectados com confidence score

## Dev Notes

### Architecture References

Todas as interfaces seguem o documento `docs/architecture/pipeline-architecture.md`.

**Pipeline Stage Interface:**

```typescript
// src/core/types.ts (definida em Story 1.1)
interface IPipelineStage<TInput, TOutput> {
  name: string;
  process(input: TInput, options?: StageOptions): Promise<TOutput>;
}

interface StageOptions {
  verbose?: boolean;
  onProgress?: (percent: number, message: string) => void;
}
```

**Pipeline Config (relevant fields):**

```typescript
// src/pipeline/runner.ts
interface PipelineConfig {
  cleaningLevel: 'light' | 'standard' | 'aggressive';
  // ...other fields
}
```

**Data Flow:**
```
StandardHTML (from Extractor) → Normalizer → NormalizedHTML → Cleaner → CleanHTML
```

Normalizer e Cleaner operam sobre strings HTML. O output deve continuar sendo StandardHTML válido.

**Preset → Cleaning Level Mapping:**
- `knowledge-base` → `light` (preservar o máximo possível)
- `rag` → `standard` (balanceado)
- `fine-tuning` → `aggressive` (dados mais limpos)

**Depends on:** Story 1.1 (interfaces, pipeline runner), Story 1.2 (extractors produzem StandardHTML)
