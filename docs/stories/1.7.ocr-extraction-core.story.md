## Story

**As a** user processing scanned PDF documents,
**I want** EXTRACTA to automatically detect scanned PDFs and extract text via OCR,
**so that** scanned documents are processed instead of being rejected with an error.

## Scope

**IN:**
- PdfOcrExtractor com Tesseract.js (local WASM)
- PdfRasterizer com pdfjs-dist + node-canvas
- TesseractOcrProvider com worker pool
- Fallback chain no ExtractorRegistry
- Testes unitários e de integração

**OUT:**
- Cloud OCR providers (Google Vision, AWS Textract) — future story
- CLI flags e configuração — Story 1.8
- Quality scoring e confidence adjustments — Story 1.9
- Image pre-processing (deskew, denoise) — future story
- Mixed PDFs (pages com texto + pages escaneadas) — future story

## Complexity
**Story Points:** 8 (L) — novas dependências nativas, worker pool, integração com pipeline existente

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| `canvas` npm requer system libs (cairo, pango) | Build falha em CI/dev sem libs | Documentar install instructions por OS; considerar `@napi-rs/canvas` como alternativa sem deps nativas |
| Tesseract.js lento em PDFs grandes (100+ pages) | UX ruim, timeout | Concurrency config + progress callback + streaming por página |
| Language data download na primeira execução | Delay inesperado | Documentar; considerar bundling eng.traineddata |
| pdfjs-dist Node.js compatibility | Import errors | Usar `legacy/build/pdf.mjs`; testar em CI |

## Acceptance Criteria

1. [ ] `PdfOcrExtractor` implementa `IExtractor` e extrai texto de PDFs escaneados via Tesseract.js

## Dev Notes

### Architecture References

Segue o documento `docs/architecture/ocr-architecture.md`.

**Fallback Chain Flow:**
```
PdfTextExtractor.extract() → ScannedPdfError
  ↓
ExtractorRegistry.extractWithFallback() catches error
  ↓
PdfOcrExtractor.extract() → rasterize → OCR → StandardHTML
```

**Key Design Decisions:**
- D1: Tesseract.js (WASM, local, zero custo) como motor OCR
- D2: Fallback chain no Registry (não wrapping) — SRP preservado
- D3: pdfjs-dist + node-canvas para rasterização (pure JS, cross-platform)

**StandardHTML Output (OCR):**
```html
<article data-source="ocr" data-confidence="78.5">
  <section data-page="1">
    <p>Texto extraído da página 1...</p>
  </section>
</article>
```

**Depends on:** Story 1.1 (interfaces), Story 1.2 (PdfTextExtractor, ExtractorRegistry)

[Source: docs/architecture/ocr-architecture.md]

### Tech Stack

- **OCR Engine:** Tesseract.js v5+ (WASM)
- **PDF Rasterization:** pdfjs-dist (Mozilla PDF.js)
- **Canvas:** node-canvas (canvas npm package)
- **Test:** Vitest

### System Requirements

- `canvas` precisa de: `cairo`, `pango`, `libjpeg`, `giflib`, `librsvg`
- macOS: `brew install pkg-config cairo pango libpng jpeg giflib librsvg`
- Ubuntu: `apt-get install build-essential libcairo2-dev libpango1.0-dev libjpeg-dev libgif-dev librsvg2-dev`

- **Fixtures:** `test/fixtures/pdf/scanned.pdf`
- **Coverage target:** >= 80%
