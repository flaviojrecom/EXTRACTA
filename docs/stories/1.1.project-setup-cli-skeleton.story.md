## Dev Notes

### Architecture References

Todas as interfaces seguem o documento `docs/architecture/pipeline-architecture.md`.

**Pipeline Flow:**
```
Input File → [Analyzer] → [Extractor] → [Normalizer] → [Cleaner] → [Structure Builder] → [Chunker] → [Exporter]
```

**StandardHTML** é o formato intermediário entre extraction e processing. Todos os extractors devem produzir HTML com subset permitido: `<article>`, `<section>`, `<h1>`-`<h6>`, `<p>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<pre>`, `<code>`, `<blockquote>`.

**Error Strategy:** Fail-fast com `ExtractaError` subclasses. Cada erro inclui `code`, `stage`, `recoverable`.

**Project Structure:**
```
src/
├── cli/commands/process.ts    # CLI command
├── core/types.ts              # IPipelineStage
├── core/document.ts           # StructuredDocument
├── core/errors.ts             # Error hierarchy
├── extractors/types.ts        # IExtractor
├── pipeline/runner.ts         # Pipeline orchestrator
├── chunker/types.ts           # ITokenizer, Chunk
├── exporters/types.ts         # IExporter
└── cache/, quality/           # Future stages
```

[Source: docs/architecture/pipeline-architecture.md]
