## Dev Notes

### Architecture References

Todas as interfaces seguem o documento `docs/architecture/pipeline-architecture.md`.

**Adapter Pattern:** Cada extractor implementa `IExtractor` e é registrado no `ExtractorRegistry` por extensão. O pipeline runner usa o registry para selecionar o extractor correto.

**Key Interfaces:**

```typescript
// src/extractors/types.ts (definida em Story 1.1)
interface IExtractor {
  supportedExtensions: string[];
  extract(input: Buffer, metadata?: FileMetadata): Promise<ExtractionResult>;
  canHandle(extension: string, buffer?: Buffer): boolean;
}

interface ExtractionResult {
  html: string;            // StandardHTML content
  metadata: DocumentMetadata;
  warnings: string[];
}

interface FileMetadata {
  fileName: string;
  extension: string;
  sizeBytes: number;
}

interface DocumentMetadata {
  title?: string;
  author?: string;
  language?: string;
  pageCount?: number;
  isScanned?: boolean;
  sourceFormat: string;
}
```

**StandardHTML** é o formato intermediário entre extraction e processing. Subset permitido: `<article>`, `<section>`, `<div>`, `<h1>`-`<h6>`, `<p>`, `<span>`, `<em>`, `<strong>`, `<a>`, `<ul>`, `<ol>`, `<li>`, `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`, `<pre>`, `<code>`, `<blockquote>`, `<br>`, `<hr>`.

**Error Strategy:** `ScannedPdfError` para PDFs escaneados, `UnsupportedFormatError` para formatos não suportados, `ExtractionError` para falhas genéricas.

**Depends on:** Story 1.1 (interfaces, project structure, error classes)
