## Story

**As a** user processing scanned PDFs,
**I want** quality scores and confidence metrics for OCR-extracted content,
**so that** I can assess the reliability of the extracted text and identify low-quality pages.

## Scope

**IN:**
- DocumentMetadata extension com campos OCR
- Confidence tracking por página
- Quality scorer adjustments para documentos OCR
- Exporter provenance OCR
- Verbose mode OCR info

**OUT:**
- Core OCR engine — Story 1.7
- CLI config — Story 1.8
- Advanced layout analysis — future story

## Complexity
**Story Points:** 5 (M) — toca em múltiplos arquivos (scorer, 4 exporters, runner, CLI)

## Dev Notes

### Architecture References

Segue o documento `docs/architecture/ocr-architecture.md`, seção 9.

**DocumentMetadata Extension:**
```typescript
interface DocumentMetadata {
  // ... existing fields ...
  isScanned?: boolean;         // true when OCR was used
  ocrConfidence?: number;      // Average confidence 0-100
  ocrEngine?: string;          // 'tesseract' | 'google-vision'
  lowConfidencePages?: number[]; // Pages below threshold
}
```

**Quality Score Impact:**
- OCR documents: `structurePreserved *= 0.6` (OCR loses headings/lists/tables)
- High confidence (>80): small bonus to metadataCompleteness
- Low confidence (<60): penalty to metadataCompleteness

**Depends on:** Story 1.7 (PdfOcrExtractor with confidence data), Story 1.8 (config)
